# Política de Seguridad - SportShop

Este proyecto ha sido desarrollado como parte de un entorno académico y no tiene fines comerciales. 
Aunque se han seguido buenas prácticas de desarrollo web en la medida de lo posible, el proyecto no 
está pensado para un uso en producción ni para entornos críticos.

## Soporte

Debido a la naturaleza académica del proyecto, no se ofrece soporte oficial ni mantenimiento continuo.
Si decides reutilizar total o parcialmente el código, se recomienda revisarlo, adaptarlo y mejorarlo,
ya que existe un amplio margen de optimización.

## Cómo reportar una vulnerabilidad

Si detectas algún problema de seguridad, se agradece que no lo publiques directamente como un *Issue*
abierto sin contexto. En su lugar, puedes seguir estas opciones:

1. **Notificación:** Abrir un [Issue de GitHub](https://github.com/jmmartineze/sportshop/issues) utilizando
   la etiqueta `security`, o usar la opción **"Report a vulnerability"** en la pestaña **Security** del
   repositorio (si está disponible).
2. **Descripción:** Incluir una descripción clara del problema, pasos para reproducirlo y el impacto
   potencial.
3. **Revisión:** El reporte será revisado cuando sea posible, teniendo en cuenta la disponibilidad
   propia de un proyecto académico.

## Compromiso de corrección

Aunque se trata de un proyecto de instituto, la calidad del código es importante. Si se confirma una
vulnerabilidad:
- Se intentará proponer una solución razonable.
- La corrección se aplicará directamente o mediante una rama separada, según el caso.
- Siempre que el usuario que haya realizado el reporte lo desee se reconocerá su labor.
